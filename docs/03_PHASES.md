# 사인전에 — Phase 분리 계획

> 한 번에 다 만들면 복잡해져서 앱인토스 검수도 떨어지고 품질이 낮아집니다.
> Phase별로 나눠서 각각 "진짜 동작하는 제품"을 만듭니다.

## Phase 1: MVP — 임대차 특화 무료 분석 (4~6주)

### 목표
임대차 계약서를 촬영하면 30초 내에 핵심 요약·위험 조항·체크리스트 3카드를 보여주는 앱. 광고 없이 무료로 제공해 초기 유저 확보.

### 기능
- [ ] 카메라 촬영 또는 갤러리에서 이미지 선택
- [ ] 클라이언트 측 개인정보 마스킹 (주민번호·주소·계좌번호 블러)
- [ ] 최초 1회 국외이전 동의 화면 (Anthropic/미국/7일 보존 명시)
- [ ] Flask 서버 → Claude Vision API 중계 (mTLS 인증)
- [ ] 임대차 계약서 특화 분석 프롬프트
- [ ] 3카드 결과 화면 (TDS UI 적용)
  - 카드 ①: 핵심 요약 (임대기간·보증금·월세·특약)
  - 카드 ②: 위험 조항 목록
  - 카드 ③: 서명 전 체크리스트
- [ ] 로컬 이력 저장 (AsyncStorage)
- [ ] 하단 고정: "AI 활용" 고지 + "법률 조언 아님" 면책 문구

### 데이터
- AnalysisSession, AnalysisResult, ConsentRecord, DailyUsage (무제한 설정)

### 인증
- 토스 계정 자동 인증 — 별도 로그인 화면 없음. `@apps-in-toss/framework` SDK가 처리.

### "진짜 제품" 체크리스트
- [ ] 실제 Claude Vision API 연결 (목업 응답 X)
- [ ] mTLS 인증서 실제 발급 후 적용 (로컬 테스트 인증서 X)
- [ ] 앱인토스 샌드박스 앱에서 실제 테스트 완료
- [ ] 앱인토스 검수 통과 (영업일 2~3일)
- [ ] 실제 계약서 10건 이상 테스트해서 결과 품질 검증

### Phase 1 시작 프롬프트

## Phase 2: 수익화 + 확장 (Phase 1 안정화 후 3~4주)

### 전제 조건
- Phase 1이 앱인토스 검수 통과 + 실제 유저 피드백 수집 완료
- 앱인토스 IAA(인앱광고) 신청 완료

### 목표
무료 분석 횟수 제한 + 광고로 수익 발생. 계약서 유형 확대로 사용 케이스 넓힘.

### 기능
- [ ] IAA 보상형 광고 연동 (Google AdMob)
  - 일 3회 무료 → 4회부터 광고 시청(30초) 후 분석 해제
- [ ] DailyUsage 제한 로직 활성화
- [ ] 결과 화면 이미지 캡처 / 공유 기능
- [ ] 계약서 유형 선택 UI 추가 (임대차 / 근로계약서 / 프리랜서)
- [ ] 근로계약서 특화 분석 프롬프트 (최저임금·주휴수당·계약해지 조항)
- [ ] 프리랜서/용역 계약서 특화 프롬프트 (저작권·대금지급·책임범위)

### 추가 데이터
- DailyUsage의 `freeQuota`를 `999` → `3`으로 변경

### 통합 테스트
- Phase 1 임대차 분석 기능이 여전히 정상 동작하는지 확인
- 광고 미시청 시 4회째 분석이 차단되는지 확인
- 앱인토스 다크패턴 정책 위반 여부 재검수

## Phase 3: 프리미엄 (IAP) — Phase 2 안정화 후

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중
- 앱인토스 IAP 신청 완료 (앱마켓 15% + 토스 5% = 20% 수수료 반영)
- 구독/정기결제 지원 여부 앱인토스 측 확인 완료

### 목표
광고 없이 쾌적하게 무제한 분석. 분석 이력 관리·고급 기능으로 유료 전환.

### 기능
- [ ] IAP 프리미엄 언락 (무제한 분석 — 광고 대체)
- [ ] 상세 분석 리포트 (PDF 내보내기)
- [ ] 분석 이력 클라우드 백업 (기기 변경 시 이력 유지)
- [ ] 계약서 2개 동시 비교 분석

### 주의사항
- IAP 수수료 20% 반영한 가격 책정 필요
- 클라우드 백업 도입 시 서버 저장 = 개인정보 처리 방침 개정 + 재동의 절차 필수
- 구독 모델은 앱인토스 공식 지원 여부 확인 후 결정
