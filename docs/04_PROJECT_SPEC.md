# 사인전에 — 프로젝트 스펙

> AI가 코드를 짤 때 지켜야 할 규칙과 절대 하면 안 되는 것.
> **이 문서를 AI에게 항상 함께 공유하세요.**

---

## 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 미니앱 프레임워크 | `@apps-in-toss/framework` (Granite) | 앱인토스 필수 요구사항, SDK 2.x 사용 (3/23 이후 1.x 불가) |
| 프론트엔드 | React Native (앱인토스 지원 스택) | 앱인토스 공식 지원 기술 스택 |
| UI 컴포넌트 | TDS (토스 디자인 시스템) | 앱인토스 필수 요구사항, 다크모드 미지원 |
| 로컬 저장 | AsyncStorage | 서버 미저장 원칙 준수, 기기 내 이력 보관 |
| 백엔드 | Python Flask | 경량 API 서버 역할만 수행 (stateless, Claude 중계) |
| AI API | Claude Vision API (`claude-sonnet-4-6` 또는 `claude-opus-4-6`) | 계약서 이미지 OCR + 분석 1회 처리 |
| 보안 통신 | mTLS (앱인토스 발급 인증서) | 앱인토스 서버 통신 필수 요구사항 |
| 패키징 | `.ait` 포맷, 100MB 이내 | 앱인토스 번들 제한 |
| 테스트 환경 | 앱인토스 샌드박스 앱 | 실제 기기 테스트 필수 |

## 절대 하지 마 (DO NOT)

> AI에게 코드를 시킬 때 이 목록을 반드시 함께 공유하세요.

- **계약서 이미지를 서버에 저장하지 마** — 1회성 처리 원칙. 서버 메모리에서 처리 후 즉시 폐기
- **분석 결과를 서버 DB에 저장하지 마** — AsyncStorage 로컬 저장만 허용
- **API 키나 비밀번호를 코드에 직접 쓰지 마** — `.env` 파일 사용, `.gitignore`에 반드시 추가
- **"법률 조언", "법률 검토", "법률 판단" 표현을 쓰지 마** — "정보 제공", "일반적 설명" 수준으로 한정
- **위험도 점수(숫자)로 계약서를 평가하지 마** — AI 판단으로 해석될 소지, 변호사법 리스크
- **"이 계약서는 불리합니다", "서명하지 마세요" 류의 단정적 표현 쓰지 마** — 면책 문구 위반
- **마스킹 전처리 없이 원본 이미지를 Claude API에 전송하지 마** — 개인정보보호법 위반
- **국외이전 동의 없이 API 호출을 허용하지 마** — ConsentRecord 확인 후에만 분석 허용
- **AI가 생성한 콘텐츠를 사람이 작성한 것처럼 표시하지 마** — Anthropic API 정책 위반
- **TDS가 아닌 커스텀 디자인을 임의로 추가하지 마** — 앱인토스 검수 탈락 원인
- **SDK 1.x를 사용하지 마** — 2026-03-23 이후 앱인토스 미지원
- **기존 AsyncStorage 스키마를 임의로 변경하지 마** — 기존 이력 데이터 손실 가능
- **다크패턴 광고를 구현하지 마** — 앱인토스 정책 위반 (진입 즉시 광고, 닫기 방해 등)

## 항상 해 (ALWAYS DO)

- **모든 Claude 응답 하단에 면책 문구 표시**: `"이 내용은 AI가 생성한 일반적인 정보입니다. 법률 조언이 아니며, 중요한 사항은 반드시 전문가와 상담하세요."`
- **결과 화면 상단에 "AI 활용" 고지**: 앱인토스 정책 의무 (위반 시 최대 3,000만원 과태료)
- **마스킹 처리 완료 여부를 사용자에게 표시**: 마스킹 완료 UI 피드백 제공
- **Claude API 오류 시 친절한 에러 메시지 표시**: "분석에 실패했습니다. 사진을 더 밝고 선명하게 찍어 다시 시도해주세요."
- **TDS 컴포넌트 우선 사용**: 커스텀 컴포넌트 최소화
- **계약서 유형이 `lease`(임대차)로 고정된 Phase 1**: 하드코딩이 아닌 `contractType` 변수로 관리
- **mTLS 인증서 경로를 환경변수로 관리**: 코드에 경로 하드코딩 금지
- **변경 전 계획 먼저 보여주기**: 주요 로직 변경 전 구현 계획 공유

## Claude Vision 분석 프롬프트 규칙

```

출력 형식:
1. 핵심 요약: [임대기간, 보증금, 월세, 주요 특약사항]
2. 주의할 조항: [불리하거나 확인이 필요한 항목들]
3. 서명 전 체크리스트: [확인해야 할 사항들]

주의사항:
- "법률 조언", "판단", "추천"이라는 표현 사용 금지

- "일반적으로", "참고로" 등 정보 제공 표현 사용
- 한국어로 쉽고 간결하게 작성
```

## 테스트 방법

```bash
# 프론트엔드 로컬 실행 (앱인토스 샌드박스 앱 필요)
cd frontend
npm install
npm start

# 백엔드 로컬 실행
cd backend
pip install -r requirements.txt
python app.py

# 마스킹 기능 단위 테스트
pytest tests/test_masking.py

# Claude API 연결 테스트 (실제 API 키 필요)
pytest tests/test_claude_integration.py
```

## 배포 방법

1. `앱인토스 개발자센터` → 워크스페이스 → 앱 등록
2. 앱인토스 Granite CLI로 `.ait` 번들 빌드: `granite build`
3. 번들 크기 확인 (100MB 이내)
4. 앱인토스 개발자센터에 `.ait` 업로드
5. 검수 신청 (영업일 2~3일)

**Flask 백엔드 배포**:
- 앱인토스 mTLS 인증서 적용 필수
- 별도 서버 또는 클라우드 (AWS/GCP) 운영

## 환경변수

| 변수명 | 설명 | 어디서 발급 |
|--------|------|------------|
| `ANTHROPIC_API_KEY` | Claude API 인증 키 | [console.anthropic.com](https://console.anthropic.com) |
| `MTLS_CERT_PATH` | 앱인토스 mTLS 클라이언트 인증서 경로 | 앱인토스 개발자센터 (7~14 영업일) |
| `MTLS_KEY_PATH` | mTLS 개인키 경로 | 앱인토스 개발자센터 |
| `FLASK_ENV` | 실행 환경 | `development` / `production` |
| `CLAUDE_MODEL` | 사용할 Claude 모델 | `claude-sonnet-4-6` (기본) |

> `.env` 파일에 저장. **절대 GitHub에 올리지 마세요.** `.gitignore`에 반드시 추가.
