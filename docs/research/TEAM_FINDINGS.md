# 발견 사항 & 공유 자료

(팀원들이 유용한 정보 발견 시 여기에 기록)

## [리서처 2] 앱인토스 수익화 방법

### 토스 자체 광고/수익화 시스템
앱인토스는 자체 광고 시스템(IAA, In-App Advertising)을 공식 지원한다.

**광고 유형:**
- **전면형 광고**: 전체화면으로 표시, 화면 전환 지점에서 노출
- **보상형 광고**: '광고 시청 → 보상 지급' 구조, 사용자가 원할 때 시청. eCPM이 가장 높은 포맷

**효과 지표:**
- ARPDAU(DAU 1명당 광고 매출), LTV 향상 가능
- 토스 측에서 광고를 자연스럽게 녹이면서 이탈을 줄이는 도구 제공

**마케팅 도구:**
- 앱 푸시, 리워드 프로모션(토스 포인트 기반), 친구 초대 리워드 기능 제공
- 데이터 대시보드, 디자인 시스템, 마케팅 솔루션 무상 제공

### 유료 전환 가능 여부
**가능하다.** 앱인토스에서 유료 서비스 운영이 가능하며, 다음 두 가지 결제 수단을 지원한다:

1. **인앱결제(IAP)**: 디지털 콘텐츠/서비스 판매용
   - 수수료: 앱마켓 15% + 토스 5% = **총 20%**
   - 향후 매출 규모 증가 시 앱마켓 수수료가 30%로 변동 가능

2. **토스페이 결제**: 실물 상품/서비스 판매용
   - 수수료: 신용·체크카드 기준 4.3% (VAT 별도)
   - 가맹점 등급(연매출)에 따라 수수료율 차등 적용

**제한 사항:**
- 현금성/환가성 상품 판매 불가
- 토스 포인트와 결합한 상품 판매 불가
- 금융 상품 중개·판매·광고 불가
- 사행성/복권/베팅성 콘텐츠 불가
- 디지털 자산/가상자산 불가

**구독/정기결제:** 앱인토스 자체적으로 구독 모델을 공식 지원한다는 명시적 문서는 확인 불가. 다만 토스페이먼츠에서 빌링키 기반 정기결제 기능을 제공하므로, 토스페이 연동 시 구독 모델 구현이 기술적으로 가능할 수 있음 (공식 확인 필요).

### 인앱결제 지원
**공식 지원한다.**
- 앱인토스 개발자센터에서 IAP(In-App Purchase) 콘솔 가이드 제공
- 인앱결제 콘솔에서 매출, 결제 사용자 데이터 추적 가능
- 앱스토어/구글플레이 인앱결제 시스템과 연동

### 실제 파트너사 비즈니스 모델 사례

**게임 분야 (전체의 약 40%):**
| 파트너사 | 서비스 | 비즈니스 모델 | 성과 |
|---------|--------|-------------|------|
| 마나바바 | 돌돌디 | IAP + IAA (게임 내 광고/결제) | 월 매출 2억 1천만원 돌파, 경영난 탈출 |
| 마나바바 | 거지키우기 | IAP + IAA | 실시간 인기 게임 1위, 누적 매출 1억 근접 |
| 위메이드커넥트 | 어비스리움 | IAP + IAA (힐링게임) | 앱인토스 미니앱 출시 |

**비게임 분야 (전체의 약 60%):**
| 파트너사 | 서비스 | 비즈니스 모델 | 성과 |
|---------|--------|-------------|------|
| 디스팟 | 할인쿠폰 | 제휴 브랜드 구매 수수료 정산 (CPA) | 5일만에 개발, 첫 달 매출 1,000만원 |
| 코심 | eSIM 해외로밍 | 직접 판매 (토스페이) | 2주 개발, 첫 주 매출 260만원 (광고 없이) |
| 안전집사 | 프롭테크(부동산) | 서비스 이용료 | 신규 고객 99% 앱인토스 경유, 월 매출 40배 성장 |
| 커리어데이 | 긱 이코노미 | 중개 수수료 | 토스와 연계한 커리어/수익 서비스 |
| 두쫀쿠 맵 | 지도 서비스 | 광고 기반 | 제휴 논의~출시 단 하루 |
| 1인 개발자 | 다수 미니앱 | IAA + IAP 혼합 | 월 매출 2,500만원 (미니앱 5개) |

**플랫폼 전체 통계 (2026년 2월 기준):**
- 제휴 미니앱 수: 1,000개 돌파 (정식 출시 7개월 만)
- 비게임 비율: 60% 이상 (일상, 건강, AI 등)
- 매출 발생 비율: 제휴 미니앱 중 75%가 실제 매출 발생
- 서비스 지속률: 95%가 현재까지 운영 중

### 결론: 현실적인 수익화 옵션

**1순위: 인앱광고(IAA) — 가장 접근성 높음**
- 진입 장벽 낮음, 별도 결제 시스템 불필요
- 보상형 광고로 eCPM 극대화 가능
- 게임·비게임 모두 적용 가능
- 토스 제공 도구로 이탈 최소화 지원

**2순위: 인앱결제(IAP) — 디지털 콘텐츠/서비스**
- 프리미엄 기능, 아이템, 디지털 콘텐츠 판매
- 수수료 20% (앱마켓 15% + 토스 5%)로, 구글/애플 독립앱 30%보다 낮음
- 게임 아이템, 프리미엄 기능 잠금 해제 등에 적합

**3순위: 토스페이 직접 결제 — 실물/서비스 판매**
- 실물 상품이나 서비스 직접 판매 시 사용
- 수수료 4.3%로 가장 낮음
- 코심(eSIM), 디스팟(할인쿠폰) 사례처럼 커머스형 서비스에 적합

**4순위: 제휴/CPA 모델 — 트래픽 기반**
- 외부 브랜드와 제휴하여 구매 전환 시 수수료 정산
- 디스팟 사례가 대표적
- 토스의 3,000만 유저 트래픽 활용

**5순위: IAA + IAP 하이브리드 — 수익 극대화**
- 기본 무료(광고) + 프리미엄 유료(IAP) 조합
- 마나바바(돌돌디) 등 게임 분야에서 가장 높은 매출 달성
- 1인 개발자도 월 2,500만원 달성 사례 있음

**핵심 장점:** 앱인토스는 초기 파트너사에게 일부 수수료 면제 혜택을 제공하며, 개발·디자인·마케팅 인프라를 무상 제공함. 3,000만 토스 유저 기반 노출이 최대 강점.

## [리서처 1] 앱인토스 플랫폼 정책

### 외부 광고 허용 여부
- **인앱광고(구글 애드몹 기반) 공식 허용** — 전면형, 보상형(리워드), 배너 광고 3가지 유형 지원
- 구글 애드몹 광고 정책을 따르며, 정책 위반 시 광고 제한/중단 가능
- 광고 수익 정산: 매월 1~31일 수익 → 다음 달 1일 업데이트 → 해당 월 말일 입금
- eCPM 기준 수익 측정 (예: eCPM 60,000원 = 노출 1회당 약 60원)
- **다크패턴 엄격 금지**: 진입 시 즉시 바텀시트/광고 표시, 예상치 못한 광고, 닫기 방해 등 금지
- "외부 광고 가이드"는 미니앱을 SNS 등 외부 채널에서 홍보할 때 "토스 미니앱" 브랜드 사용 기준 (미니앱 내부 광고와는 별개 개념)
  - 허용 표현: "[서비스명]을 '토스 미니앱'으로 만나보세요"
  - 금지 표현: "토스 공식 파트너사", "토스 입점사", "토스 제휴사"
  - 신규 캠페인/보도자료 전 검수 필수

### 수수료 정책
- **서비스 수수료**: 공식 페이지에 "지금은 서비스 수수료가 없어요"라고 명시 (초기 파트너 면제 프로모션)
- **인앱광고 수수료**: 정상 15%, CBT/테스트 기간 0%
- **인앱결제(IAP) 수수료**: 토스 자체 5% (CBT 기간 0%) + 앱스토어/구글플레이 15~30% 별도
  - 결제 가능 금액 범위: 400원 ~ 1,400,000원
- **토스페이 결제 수수료**: 신용/체크카드 기준 4.3% (VAT 별도), 가맹점 등급별 차등
- 프로모션 무료 기간의 정확한 종료일은 공식 문서에서 명확히 확인 불가 (이전에 2026년 2월 28일까지라는 정보 존재, 현재 공식 페이지에는 "지금은 없다"로만 표기 — 연장되었거나 상시 무료로 전환된 것으로 추정되나 확정 불가)

### 개발 제약사항
- **지원 기술 스택**: WebView, React Native, Cocos Creator (게임용)
- **필수 프레임워크**: Granite 기반 스캐폴딩 필수 (기존 RN 프로젝트도 재구성 필요)
- **필수 패키지**: `@apps-in-toss/framework`
- **SDK 요구사항**: 2026년 3월 23일 이후 SDK 1.x 번들 업로드 불가 → SDK 2.x 마이그레이션 필수
- **번들 크기 제한**: 100MB (비압축 기준)
- **패키징 형식**: `.ait` 확장자
- **라우팅**: Next.js 유사 파일 기반 라우팅 (`intoss://` 스킴)
- **디자인**: TDS(토스 디자인 시스템) 사용 필수 (비게임 미니앱), 로컬 브라우저 테스트 불가 → 샌드박스 앱 필수
- **다크모드**: 미지원
- **서버 API 통신**: mTLS(양방향 인증) 인증서 필수, 키 발급 7~14 영업일
- **외부 링크**: 서비스 무관 외부 링크 금지 (법률 고지, 공공기관 연결 등은 허용)
- **자사 앱 설치 유도 금지**
- **오픈 제한 서비스**: 디지털 자산/가상자산(NFT), 자금세탁 우려, 사행성/복권/베팅, 금융 상품 중개/판매/광고, 투자 자문/리딩방, 소셜/데이팅 플랫폼
- **생성형 AI 서비스**: 가능하나 "AI 활용" 고지 및 "AI 생성" 표시 의무 (위반 시 최대 3,000만 원 과태료)
- **어뷰징 방지**: 핵심 기능 동일한 미니앱 중복 등록 불가, 동일 브랜드 중복 출시 금지

### 파트너 신청 방법 및 심사 기준
**신청 절차:**
1. **회원가입**: 만 19세 이상, 본인 명의 토스앱 필요 (토스 비즈니스 회원 기반)
2. **워크스페이스 생성**: 사업자당 1개, 멤버 초대 가능 (관리자/구성원 권한 구분)
3. **사업자 등록** (수익화 기능 사용 시 필수):
   - 개인사업자: 사업자등록증 제출
   - 법인사업자: 사업자등록증 + 등기부등본(3개월 이내)
   - 대리인 신청 시: 인감증명서 + 위임장 + 신분증 추가
   - **부가가치세 면세 사업자는 등록 불가**
   - 사업자등록증 업종과 미니앱 서비스 업종 일치 필수
4. **앱 등록**: 기본정보, 카테고리, 노출 정보 (개발 미완료 시에도 사전 등록 가능)
   - 앱 로고: 600x600px PNG, 배경색 필수 (투명 배경 불가)
   - appName: 등록 후 수정 불가

**심사 절차:**
- 사업자 검토: 영업일 1~2일
- 앱 검수 (4단계): 운영 → 기능 → 디자인 → 보안 (영업일 2~3일)
- 게임 앱: 게임물관리위원회 등급분류 심의 추가 필수
- 검수 결과 이메일 안내 → 승인 후 콘솔에서 '출시하기' 클릭

**사업자 등록 없이 사용 가능한 기능:**
- 세그먼트, 푸시/알림, 게임 프로필 및 리더보드, 공유 리워드

### 시장 특징 요약

1. **B2B 편중**: 국내 계약서 분석 AI 서비스는 거의 전부 기업 법무팀 대상(B2B). 개인/소규모 사업자가 쉽게 쓸 수 있는 서비스가 사실상 없음
2. **가격 진입장벽**: 최소 월 수만원~수십만원, 무료 체험은 극히 제한적
3. **모바일 앱 부재**: 대부분 웹 기반. 모바일 앱으로 계약서를 촬영/분석하는 서비스는 로폼(Google Play) 정도가 유일하나 문서 작성 중심
4. **"분석/요약" 특화 부재**: 기존 서비스는 계약관리(CLM), 전자서명, 문서작성에 포커스. "사진 찍으면 핵심 요약" 같은 간편 분석은 없음
5. **변호사법 이슈**: 로톡 사태(2024)처럼 AI 법률 서비스가 변호사법 위반 논란에 휘말릴 수 있음 → 규제 리스크 존재

### 앱인토스 내 법률/계약 관련 서비스

- **현재 확인된 법률/계약 분석 서비스: 없음**
- 앱인토스 카테고리: 쇼핑, 게임, 컨텐츠, 유틸리티 등이 확인됨. 법률 전용 카테고리는 별도로 존재하지 않는 것으로 보임
- 앱인토스 커뮤니티 및 검색 결과에서 법률 관련 미니앱 입점 사례를 찾지 못함
- **계약서 분석/요약 앱으로 앱인토스에 입점하면 법률 카테고리의 선점 효과를 누릴 수 있음**

### 차별화 포인트 (우리만의 경쟁 우위)

1. **개인/소규모 사업자 타겟**: 기존 서비스가 모두 B2B인 반면, 일반인이 계약서 사진만 찍으면 즉시 요약해주는 B2C 서비스는 공백 상태
2. **모바일 퍼스트**: 카메라로 계약서 촬영 → AI 즉시 분석/요약. 기존 경쟁사는 PC 웹 기반
3. **무료/저가 진입**: 기존 서비스 최소 월 1~10만원 vs 무료 또는 건당 소액 과금 가능
4. **앱인토스 선점**: 토스 3,000만 사용자 대상, 법률/계약 카테고리에 경쟁 서비스 전무
5. **간편한 UX**: 복잡한 CLM/법무관리 기능 없이, "핵심 조항 요약 + 위험 조항 경고" 딱 2가지에 집중
6. **생활밀착형**: 부동산 임대차, 근로계약서, 프리랜서 계약서 등 일반인이 자주 접하는 계약서 템플릿 특화

## [리서처 4] Claude Vision API 법적 리스크

### 개인정보보호법(PIPA) 관련

**핵심 이슈: 계약서 이미지를 해외 AI API로 전송하는 행위는 "개인정보 국외이전"에 해당한다.**

1. **위탁 vs 제3자 제공 구분 (가장 중요한 판단 기준)**
   - Anthropic API가 학습 데이터로 사용하지 않는 경우 → **처리위탁(위탁)**에 해당
   - Anthropic API가 학습 데이터로 사용하는 경우 → **제3자 제공**에 해당
   - Anthropic API는 기본적으로 상업용 API 입력/출력을 모델 학습에 사용하지 않음 → **위탁으로 분류 가능성 높음**

2. **개인정보보호법 적용 조항**
   - 제17조(개인정보의 제공): 제3자 제공 시 정보주체 동의 필요
   - 제26조(업무위탁에 따른 처리 제한): 위탁 시 문서에 의한 계약, 수탁자 관리감독 의무
   - 제28조의8(국외이전): 국외 이전 시 정보주체에게 사전 고지 + 동의 필요

3. **필수 고지사항 (국외이전 시)**
   - 이전되는 개인정보 항목
   - 이전 국가, 시기, 방법
   - 수령자(Anthropic) 신원 및 연락처
   - 이용 목적 및 보유기간
   - 거부 절차 및 불이익

### 1회성 처리/서버 미저장 방식의 법적 유효성

1. **Anthropic API 데이터 보존 정책**
   - API 입력/출력: 모델 학습에 사용하지 않음 (기본 정책)
   - API 로그 보존: 2025년 9월 14일 이후 **7일간만** 보존 후 자동 삭제
   - Zero Data Retention(ZDR) 계약 가능: 엔터프라이즈 고객은 입력/출력 미저장 계약 가능
   - 단, 안전 분류기(User Safety classifier) 결과는 정책 집행을 위해 보존

2. **법적 유효성 판단**
   - "1회성 처리, 서버 미저장"이라고 해도 **전송 자체가 국외이전에 해당**하므로 동의 의무는 면제되지 않음
   - 다만, 보존기간이 극히 짧다는 점(7일)은 위험도 완화 요소로 작용
   - ZDR 계약 체결 시 법적 방어력 강화 가능
   - **전문가 확인 필요**: "1회성 API 호출이 국외이전에 해당하는지"에 대한 명확한 판례나 행정해석은 아직 확인되지 않음

### 민감 정보 포함 시 대응

1. **민감정보 해당 항목 (개인정보보호법 제23조)**
   - 주민등록번호, 여권번호, 운전면허번호 등 고유식별정보
   - 건강정보, 성생활, 사상/신념, 노동조합 가입 등
   - 계약서에 포함 가능: 주민번호, 주소, 연락처, 계좌번호

2. **대응 방안**
   - **사전 비식별화(마스킹)**: 이미지 전송 전 주민번호, 주소 등을 마스킹 처리 → 가장 안전한 방법
   - **별도 동의**: 민감정보/고유식별정보 처리에는 **별도의 명시적 동의** 필요
   - **최소 수집 원칙**: 분석에 불필요한 개인정보는 수집하지 않도록 설계
   - **주민등록번호**: 원칙적으로 처리 금지 (법령에 구체적 근거 있는 경우만 예외)

### Anthropic API 약관 주요 제한사항

1. **법률 서비스 관련**
   - "법률 해석, 법률 지침, 또는 법적 영향이 있는 결정"에 사용 시 **반드시 human-in-the-loop(전문가 검토)** 필수
   - AI 결과를 사람이 생성한 것처럼 제시하는 것 금지

2. **개인정보 관련**
   - 프라이버시 권리 침해 금지
   - 비공개 개인정보(연락처, 건강정보, 생체정보 등) 무단 수집/접근 금지
   - 생체 데이터 분석으로 인종/종교 등 추론 금지

3. **데이터 활용 관련**
   - API 입력/출력을 AI 모델 학습에 사용하는 것(모델 스크래핑, 증류) 금지 (사전 허가 없이)
   - 고위험 분야(의료, 금융, 보험, 고용/주거, 학술시험)에서는 인간 검토 + AI 사용 사실 공개 필수

4. **위반 시 제재**
   - 접근 제한(throttle), 정지(suspend), 종료(terminate) 가능

### 변호사법 이슈 (법률 조언 vs 정보 제공 경계)

1. **2025년 5월 서울행정법원 판결 (핵심 선례)**
   - AI 고소장/법률문서 **자동작성** 서비스: **변호사법 위반 아님**
   - 판단 기준: "구체적/개별적 사실관계 파악, 법규 검토, 법적 추론/평가 요소가 가미되어 있는지"
   - **자동작성(허용)**: 사용자 입력 내용이 수정 없이 그대로 반영되는 형식
   - **검토 서비스(위반 가능성)**: 구체적 사실관계를 검토/수정하는 과정이 포함

2. **계약서 분석 앱에 대한 적용**
   - 계약서 조항을 일반적으로 설명하는 수준 → **정보 제공** (변호사법 위반 아님 가능성)
   - 특정 계약서의 유/불리를 판단하고 수정 권고 → **법률 조언** (변호사법 위반 가능성)
   - 핵심: **"구체적 사건에 대한 법적 판단"을 제공하느냐** 여부

3. **무료 vs 유료**
   - 리걸테크 전문 변호사들: "완전 무료라면 문제 없다"는 의견 다수
   - 유료 서비스라면 변호사법 위반 리스크 증가

4. **권장 표현 방식**
   - "이 분석은 일반적인 정보 제공 목적이며, 법률 조언이 아닙니다"
   - "구체적 법률 문제는 변호사와 상담하세요" 면책 문구 필수

### 위험도 평가

- **개인정보 이슈: 상** — 계약서에는 거의 반드시 개인정보(이름, 주소, 주민번호 등)가 포함됨. 해외 서버 전송은 국외이전에 해당하며, 동의 없이 진행하면 개인정보보호법 위반. 과징금/형사처벌 가능.
- **변호사법 이슈: 중** — 서비스 설계에 따라 달라짐. "일반 정보 제공" 수준으로 설계하면 위험 낮음. "법률 조언/검토" 수준이면 위반 가능성 높음. 2025년 판결이 자동작성에 대해 우호적 판단을 내렸으나, 계약서 "분석"은 별개 판단 필요.
- **Anthropic 약관 이슈: 중** — 법률 관련 사용 자체가 금지는 아니나, human-in-the-loop 미구현 시 약관 위반으로 서비스 정지 가능. 개인정보 무단 처리 시에도 위반.

### 권장 완화 방법

1. **개인정보 마스킹 기능 필수 구현**: 이미지 전송 전 주민번호, 주소, 계좌번호 등 자동 마스킹 (OCR + 정규식 탐지)
2. **이용약관/개인정보처리방침에 국외이전 명시**: Anthropic API 사용 사실, 데이터 이전 국가(미국), 보존기간(7일) 등 고지
3. **정보주체 동의 획득 절차 구현**: 앱 내에서 계약서 업로드 시 동의 화면 표시
4. **면책 문구 적용**: "본 서비스는 법률 조언이 아닌 일반적 정보 제공입니다" 명시
5. **human-in-the-loop 안내**: "중요한 법률적 판단은 반드시 변호사와 상담하세요" 권고
6. **ZDR(Zero Data Retention) 계약 검토**: Anthropic과 데이터 미저장 계약 체결 고려
7. **서비스 범위 제한**: "법적 판단/추론"이 아닌 "조항 요약/일반 설명" 수준으로 기능 한정
8. **처리 위탁 계약서(DPA) 체결**: Anthropic과 개인정보 처리 위탁에 관한 문서화된 계약 확보
